**NPC to FinalBoss In ML**

Everyone starts as an NPC. This repo documents my grind from ML noob to final boss level: the concepts I've learned, the code I've broken (and fixed), and the projects I've built along the way. No skips, no cheat codes, just consistent levelling up.

## Who this is for

This repo tracks my journey toward becoming an **AI/ML Engineer** (not a Data Scientist, not a research-level AI Scientist).

- **AI/ML Engineer** = building and shipping products using ML (models, RAG systems, AI features): this is the goal here.
- **Data Scientist** = business analytics, insights from data (different track).
- **AI Scientist/Researcher** = inventing new ML methods, PhD-level research (a much later, different goal).

This repo covers foundations through applied/portfolio-level skills: Python and math basics, classical ML algorithms, deep learning, transformers and LLMs, RAG, and deploying real projects. It's aimed at getting internship and entry-level AI/ML engineer job ready, not research-track.

### What an AI/ML Engineer actually does

- **Builds models for real problems**: picks between using an existing pretrained model (e.g. from Hugging Face) or training one from scratch (classical ML or deep learning), depending on the problem.
- **Builds AI products/features**: integrates LLMs (OpenAI, Claude, open-source models) into real apps. Example: a support chatbot that answers from a company's own docs (this is RAG).
- **Builds data pipelines**: turns raw, messy data into something a model can actually use, often automated so new data flows in without manual work.
- **Deploys models to production**: wraps a model in an API and connects it to a real app, so real users can use it, not just a notebook running on a laptop.
- **Monitors and improves models after launch**: data changes over time, so accuracy can drift. Part of the job is catching that and retraining when needed.

In short: an AI/ML Engineer doesn't just build a model, they build the whole system around it: get the data, build the model, fit it into a real product, keep it running. That's why this repo covers both ML (building the model) and the software/DSA side (making it fast and deployable), since both are needed to be job ready.
